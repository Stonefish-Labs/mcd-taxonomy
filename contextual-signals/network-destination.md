# Network Destination Signals

| Signal | Description | Effect |
|---|---|---|
| **Sanctioned or embargoed jurisdiction destinations** | Network endpoints resolving to IP ranges in OFAC-sanctioned countries, EU-sanctioned jurisdictions, or other restricted regions. This signal amplifies any `NETW.*` finding — a library making undocumented HTTP calls is suspicious; one making undocumented calls to sanctioned jurisdictions demands immediate investigation. Note that geolocation databases have accuracy limitations; confirm with multiple sources before treating jurisdiction as definitive. | High amplifier on any `NETW.*` finding — triggers compliance review in addition to security investigation |
| **Bulletproof or non-cooperative hosting infrastructure** | Destination IP ranges associated with hosting providers known for ignoring abuse complaints and law enforcement takedown requests. These providers are disproportionately used for C2 infrastructure, phishing, and data collection because the attacker's infrastructure persists longer. | Medium-high amplifier — increases confidence that the destination is attacker-controlled |
| **Recently registered domains** | Destination domains with WHOIS registration dates within 30-90 days of the package's publication date. Legitimate services use established domains; attacker C2 infrastructure is typically registered shortly before or concurrently with the campaign launch. Registration date correlation with package publication is a strong signal. | High amplifier — temporal correlation between domain registration and package publication is characteristic of coordinated campaigns |
| **Privacy-proxied or anonymized registration** | Destination domains behind WHOIS privacy services, registered with anonymized contact information, or using registrars popular with threat actors. While privacy services are increasingly common for legitimate domains, their presence combined with other adverse signals (recent registration, unusual TLD, no web presence) increases confidence in malicious intent. | Low amplifier in isolation — many legitimate domains use privacy services. Medium when combined with other adverse destination signals |
| **Dynamic DNS and tunneling services** | Destinations using services like `*.duckdns.org`, `*.ngrok.io`, `*.trycloudflare.com`, `*.serveo.net`, `*.localtunnel.me`, or other dynamic DNS and tunnel providers as apparent production infrastructure. These services provide free, disposable endpoints that require no identity verification — ideal for temporary C2 but inconsistent with legitimate production services. | High amplifier — dynamic DNS endpoints as a dependency's network destination have almost no legitimate explanation |
| **Residential or mobile IP ranges** | Destination IPs resolving to residential ISP or mobile carrier address space rather than datacenter or cloud infrastructure. Indicates either a compromised residential host being used as a relay, or a VPN/proxy endpoint for attribution evasion. Legitimate SaaS and API endpoints do not operate from residential IPs. | Medium amplifier — residential IP destinations are anomalous for any documented service endpoint |
| **IP-literal destinations (no DNS)** | Network calls using raw IP addresses instead of domain names. Bypasses DNS-based monitoring and domain reputation checks. While some internal infrastructure uses IP literals, public-facing services accessed by third-party libraries use DNS names. IP-literal destinations in a dependency's outbound calls are a strong evasion indicator. | Medium-high amplifier — IP literals bypass a major monitoring and reputation layer |
